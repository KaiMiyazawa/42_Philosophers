# Philosophers

42 School の哲学者の食事問題を解く課題です。マルチスレッドとミューテックスを使って、デッドロックとデータ競合を避けながらシミュレーションを動かします。

## 概要

- 出力物: `philo`
- 主要要素: `pthread`, mutex, monitor thread
- 目的: 死亡監視、食事回数制御、同期の整理

## 特徴

- 哲学者ごとのスレッドと監視スレッドを分離
- フォーク取得順を非対称にしてデッドロックを回避
- 出力用ミューテックスでログの混線を防止

## ビルド

```bash
cd philo
make
make clean
make fclean
make re
```

## 実行例

```bash
./philo 5 800 200 200
./philo 4 410 200 200 7
./philo 1 200 100 100
```

## 実装メモ

- `actions.c` で哲学者の動作を実装
- `simu.c` でシミュレーション制御
- `monitor.c` で死亡と終了条件を監視
- `init.c` で構造体とミューテックスを初期化
